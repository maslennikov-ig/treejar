## 2\. Требования к функционалу бота

**2.1. Основные функции:**

| Функция | Описание | Критерии приемки |
| :---- | :---- | :---- |
| **Определение статуса клиента** | Бот определяет статус клиента: Новый или Существующий.  | При поступлении первого запроса от клиента в диалоге бот корректно определяет статус клиента: Новый или Существующий. Вносит новых клиентов в CRM систему (метод: по API Zoho CRM). |
| **Мультиязычная работа** | Бот распознаёт и передаёт информацию на двух языках: Английский и Арабский. | Бот распознаёт информацию на двух языках: Английском и Арабском, Предоставляет ответ на языке обращения (одном из двух указанных выше). |
| **Распознавание голосовых сообщений** | Бот распознаёт голосовые сообщения и отвечает на них. | Бот распознаёт голосовые сообщения на арабском и английском языках и отвечает на них на языке обращения. |
| **Консультирование пользователей** | Консультирование пользователей по ассортименту товаров компании. | Консультирование пользователей по актуальному ассортименту товаров компании, их цене, характеристикам, срокам доставки. (Данные предоставляются по API сайта заказчика или через изменяемую google таблицу). При консультации чат-бот руководствуется Правилами ведения диалога ([https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF\_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.ewy6uvjcy26j](https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.ewy6uvjcy26j))  |
| **Реализация продажи товара**            | Реализация продажи товара исходя из приложенных правил ведения диалога. | Чат-бот реализовывает продажу товара в соответствии с приложенными правилами ведения диалога и указанными в них методиками. ([https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF\_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.ewy6uvjcy26j](https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.ewy6uvjcy26j))Информацию о ценах, остатках и описании товаров чат-бот берёт либо из изменяемой GOOGLE таблицы, которую предоставляет заказчик, либо по API Zoho inventory. |
| **Работа с возражениями** | Работа с возражениями клиента в соответствии с приложенными правилами и ценностями компании.  | Бот прорабатывает возражения клиентов руководствуясь методиками и ценностями компании, описанными в приложении.При работе используются правила ведения диалога ([https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF\_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.ewy6uvjcy26j](https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.ewy6uvjcy26j)) и приложение о ценностях компании ([https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF\_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.v5l1b2ws90v2](https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.v5l1b2ws90v2))  |
| **Предоставление клиенту статуса заказа** | При наличии заказа предоставление статуса (подтверждение, доставка, оплата) после запроса.  | Бот по запросу предоставляет корректный статус заказа. Статус заказа бот получает по API Zoho CRM или по API сайта заказчика по дальнейшей договорённости. |
| **Напоминания об оплате. «Дожимы»** | Бот производит дожимы при несогласовании Sales Order. Отправляет напоминания об оплате при неуплате товара. | Бот производит дожимы при несогласовании Sales Order согласно правилам ведения диалога и описанной стратегии. Отправляет напоминания об оплате при неуплате товара.Правила ведения диалога: ([https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF\_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.ewy6uvjcy26j](https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.ewy6uvjcy26j))  |
| **Сбор обратной связи** | После закрытия сделки бот собирает обратную связь о качестве доставки и товаре. | После закрытия сделки бот собирает обратную связь о качестве доставки и товаре.Правила ведения диалога: ([https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF\_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.ewy6uvjcy26j](https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.ewy6uvjcy26j))  |
| **Рассчёт стоимости заказа** | Актуальный рассчёт стоимости заказа (товар+доставка) с учётом скидок и наценок. | Бот производит актуальный рассчёт стоимости заказа (товар+доставка) с учётом скидок и наценок, руководствуясь прайс-листом и предоставленными сведениями о скидках и наценках.Стоимость товаров и доставки бот получает по API Zoho inventory или через изменяемую google таблицу. |
| **Создание карточки сделки в CRM системе** | Создание и заполнение (имя, телефон, email, источник,UTM метка) карточки сделки в CRM системе. | Бот создаёт и корректно заполняет карточку сделки в CRM системе. Используется API Zoho CRM. |
| **Передвижение сделок в воронке** | Передвижение сделок (по алгоритму «Новый Deals» → «В работе» → «Согласование» → «Оплачен» ) в CRM системе. | Бот корректно передвигает сделки в CRM системе по указанному алгоритму и присваивает верный статус сделки. Используется API Zoho CRM. |
| **Сохранение истории переписки** | Чат бот сохраняет историю переписки с клиентами в базе данных. | В базе данных отображается история переписки чат-бота с клиентом. Используется API внутренней векторной базы данных. |
| **Оценка качества работы чат-бота продавца-консультанта** | Настройка бота контроля качества, оценивающего работу чат-бота продавца-консультанта по алгоритму. | Бот верно оценивает работу ИИ продавца в соответствии с методикой оценки качества ([https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF\_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.7miz803kzgbs\#heading=h.p8owhs9fxs2k](https://docs.google.com/document/d/1wPOL3H4zg1qwyJ33FF_kLzICI7nTpEpnVUialJ-JXuA/edit?tab=t.7miz803kzgbs#heading=h.p8owhs9fxs2k))  |
| **Предоставление отчётности по работе ИИ продавца** | Бот контроля качества по запросу предоставляет отчётность о работе чат-бота продавца-консультанта. | Бот контроля качества по запросу предоставляет отчётность о работе чат-бота продавца-консультанта. |
| **Предоставление отчётности по работе продавцов** | Бот контроля качества по запросу предоставляет отчётность о работе продавцов исходя из анализа диалогов. | Бот контроля качества анализирует диалоги продавцов и по запросу предоставляет отчётность о их работе. |
| **Предоставление фотографий товара**  | ИИ-продавец предоставляет фотографии товаров, о которых идёт речь, при запросе клиентом. | Бот при запросе предоставляет клиенту фото товара, о котором идёт речь. Фото забираются из сайтов или файлов заказчика. |
| **Создание базы данных** | Создаётся внутренняя база данных для бота. | Создаётся внутренняя база данных для бота с информацией из следующих источников: API Zoho Inventory, API Zoho CRM, Google таблица с прайс листом, сайты [http://treejartrading.ae/ksa-en](http://treejartrading.ae/ksa-en) и [http://bazara.ae](http://bazara.ae) (CSV файл) \- google таблица (фото товара в виде ссылок). |

**2.2. Дополнительные требования:**

* Техническая возможность модернизации чат-бота в будущем.

* Предоставить возможность редактирования промптов чат-бота. (При условии снятия ответственности за работу чат-бота с исполнителя).
