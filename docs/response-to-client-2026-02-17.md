## 4. Рекомендации по репозиторию

### 4.1. Безопасность: файл `.env.keys`

В репозитории закоммичен файл `.env.keys` с реальными API-ключами (Zoho, Wazzup, Shopify). Хотя репозиторий приватный, это не лучшая практика по нескольким причинам:

- **Git-история хранит всё навсегда.** Даже если удалить файл позже, ключи останутся в истории коммитов.
- **Добавление collaborator'ов.** Каждый, кого вы пригласите в репозиторий, получит доступ ко всем ключам.
- **Случайная публикация.** Если репозиторий когда-либо станет публичным — ключи окажутся в открытом доступе.

**Рекомендация:** Передавать ключи через защищённый канал (менеджер паролей, зашифрованное сообщение). Мы уберём `.env.keys` из репозитория и добавим его в `.gitignore` при настройке проекта. Ключи будут храниться только на сервере в `.env` файле.
