## 🔗 Что нужно от собственника

### Действие 1: Настроить Webhook в Wazzup (обязательно)

Это одноразовая операция, которая «подключит» бот к WhatsApp.

1. Войдите в **личный кабинет Wazzup** → [app.wazzup24.com](https://app.wazzup24.com)
2. Перейдите в **Настройки → Интеграции** (или **Settings → Integrations**)
3. Найдите раздел **«Webhook»** или **«API»**
4. Впишите наш URL для приёма сообщений:
   ```
   https://noor.starec.ai/api/v1/webhook/wazzup
   ```
5. Убедитесь, что подписка включена для типа **«Новые сообщения и статусы»** (Messages and Statuses)
6. Сохраните настройки

> **Альтернативный способ через API (для продвинутых):**
> Если в ЛК нет визуальной настройки webhook, его можно настроить через API-запрос:
> ```bash
> curl -X PATCH "https://api.wazzup24.com/v3/webhooks" \
>   -H "Authorization: Bearer ВАШ_API_КЛЮЧ" \
>   -H "Content-Type: application/json" \
>   -d '{
>     "webhooksUri": "https://noor.starec.ai/api/v1/webhook/wazzup",
>     "subscriptions": {
>       "messagesAndStatuses": true,
>       "channelsUpdates": true
>     }
>   }'
> ```
> При подключении Wazzup отправит тестовый запрос `{test: true}` — наш сервер ответит `200 OK`.

## 🧪 Варианты тестирования (выберите один)

### Вариант А: Тестовый WhatsApp-номер (Рекомендуемый ✅)

**Суть:** Подключить к Wazzup ещё один номер (SIM-карту), который будет использоваться **только для тестирования** бота. Основной номер продаж продолжает работать как обычно с менеджерами.

**Плюсы:**
- Полная изоляция — клиенты не пострадают
- Можно тестировать сколько угодно, без ограничений
- Нет риска случайного ответа реальному клиенту

**Минусы:**
- Нужна дополнительная SIM-карта
- На тарифе Max есть лимит на количество каналов (сейчас 2 из 5)

**Что нужно от вас:**
1. Купить SIM-карту ОАЭ (или использовать любой свободный номер)
2. Подключить её в Wazzup как новый канал (Settings → Channels → Add Channel → WhatsApp)
3. Сообщить нам номер этого тестового канала

---

### Вариант Б: Белый список на рабочем номере (Компромиссный 🟡)

**Суть:** Бот подключается к **основному** WhatsApp-номеру продаж, **НО отвечает только на сообщения от конкретных тестовых номеров** (ваш, наш, тестировщика). Все остальные сообщения от реальных клиентов бот игнорирует — они обрабатываются менеджерами как раньше.

**Плюсы:**
- Не нужна дополнительная SIM-карта
- Тестируем в реальных условиях
- Можно легко переключиться на полный режим, когда будем готовы

**Минусы:**
- Есть (минимальный) риск, что бот получит чужие сообщения (но не ответит на них)
- Менеджеры увидят, что кто-то пишет на номер (тестовые сообщения)

**Что нужно от вас:**
1. Настроить webhook (Действие 1 выше)
2. Прислать нам список номеров WhatsApp для белого списка (ваш, тестировщика)
3. Мы добавим их в настройки бота через Админ-панель

---

### Вариант В: Полный запуск на рабочем номере (Только после тестирования ⚠️)

**Суть:** Бот отвечает **всем** клиентам. Только после полного тестирования (варианты А или Б) и вашего подтверждения.
