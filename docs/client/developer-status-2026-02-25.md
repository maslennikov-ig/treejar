## Сервер для деплоя (п.6.1)

Отдельный VPS покупать **не нужно**. У нас есть выделенный сервер Hetzner с огромным запасом ресурсов:

**Хост:** `136.243.71.213` (Германия)
- CPU: AMD Ryzen 9 7950X3D — 16 ядер / 32 потока
- RAM: 128 GB DDR5 ECC
- Диск: 2 × 1.92 TB NVMe SSD

**Docker-контейнер для TreeJar уже создан** — PostgreSQL работает на порту 5435.

**Что нужно сделать на сервере:**
1. Расширить лимиты контейнера TreeJar: сейчас 2 vCPU / 2 GB → нужно **4 vCPU / 8 GB**
2. Добавить сервисы в docker-compose: **Qdrant** (6333), **Redis** (6379), **FastAPI** (8000)
3. Настроить nginx reverse proxy: **noor.starec.ai** → контейнер
4. Let's Encrypt SSL для HTTPS

PostgreSQL уже есть (порт 5435, база `treejar`, юзер `treejar`). Ресурсов хватает с большим запасом — проект Noor займёт менее 10% мощности сервера.

SSH-доступ к серверу и контейнеру предоставим при старте.

## Что сделано НЕ по плану и почему

**П.6.3 — LLM провайдер:** в изначальном плане был **OpenRouter** (прокси-агрегатор). Заменили на **DeepSeek API напрямую**. Причина: OpenRouter — посредник с наценкой 10-20% без доп. ценности для нашего проекта. DeepSeek API полностью совместим с форматом OpenAI, работает напрямую, дешевле, стабильнее. Модель та же — DeepSeek V3.2. API-ключ создан, проверен, баланс ~$10 для тестирования. Переход на DeepSeek позволяет отказаться от дорогой связки моделей (типа Claude Sonnet) в пользу одной мощной и дешевой.

**П.6.1 — VPS:** в плане предполагалась покупка отдельного VPS (4 CPU, 8 GB, ~3000 ₽/мес). Вместо этого используем существующий выделенный Hetzner-сервер — мощнее, бесплатно (уже оплачен), контейнер уже создан. Доп. расходов нет.
