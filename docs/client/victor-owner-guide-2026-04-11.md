# Виктор: Что Уже Реализовано И Что Можно Проверить

Дата: `11 апреля 2026`
Среда: production `https://noor.starec.ai`

## Что уже работает

- Бот умеет проходить точный quotation flow по реальному товару и количеству.
- Для quotation используется реальная Zoho-метадата, а не generic `DRAFT`.
- Менеджерское согласование quotation приходит в админский Telegram.
- После подтверждения клиент получает PDF quotation в WhatsApp.
- В PDF уже есть картинка товара, а не только текст.
- Кнопки manager review после deploy продолжают работать.
- Ответ менеджера из Telegram снова уходит клиенту даже для synthetic smoke threads.
- Режим `FAQ + клиент` не должен засорять базу знаний контекстными или одноразовыми ответами: такие ответы автоматически downgrade’ятся в private-only.

## Что Виктор может посмотреть в Telegram

- Сообщения manager review с inline buttons.
- Служебные сообщения после действий менеджера:
  - подтверждение/отклонение заказа
  - подтверждение отправки ответа клиенту
  - предупреждение, если ответ отправлен клиенту, но не добавлен в общую базу знаний

## Безопасные сценарии наблюдения

### 1. Наблюдение за quotation review

- Дождаться или попросить controlled test, где клиент запрашивает точную цену/quotation по конкретному SKU и количеству.
- В админской Telegram-группе должен появиться review с кнопками подтверждения/отклонения.
- После подтверждения клиент должен получить PDF quotation и подтверждающий текст.

### 2. Наблюдение за manager reply

- На manager escalation сообщении выбрать `👤 Только клиенту`.
- Ввести текст ответа в той же Telegram-группе.
- Бот должен показать подтверждение, что ответ отправлен клиенту.

### 3. Наблюдение за safe `FAQ + клиент`

- На тестовом escalation сообщении выбрать `📚 FAQ + клиент`.
- Если ответ выглядит контекстным, одноразовым, time-specific или callback-style, бот должен:
  - отправить ответ клиенту
  - не добавлять его в общую Базу Знаний
  - показать предупреждение, что ответ был downgraded to private-only

## Что лучше не делать без координации

- Не жать `FAQ + клиент` на реальных клиентских диалогах ради эксперимента.
- Не подтверждать/отклонять реальные коммерческие предложения без понимания контекста.
- Не использовать случайные тестовые тексты на живых клиентских потоках.

## Что считать хорошим результатом

- В Telegram видно корректный manager review поток.
- После действий менеджера бот даёт понятное служебное подтверждение.
- Клиентские сообщения реально доходят.
- Контекстные тестовые ответы не засоряют общую базу знаний.

## Если что-то выглядит странно

Сообщить команде:

- пример сообщения в Telegram
- время по Москве
- что именно нажали
- что ожидали увидеть

Этого достаточно, чтобы быстро поднять server-side logs и восстановить цепочку событий.
