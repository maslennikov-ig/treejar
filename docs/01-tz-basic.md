## 2\. Требования к функционалу бота

**2.1. Основные функции:**

| Функция | Описание | Критерии приемки |
| :---- | :---- | :---- |
| **Определение статуса клиента** | Бот определяет статус клиента: Новый или Существующий.  | При поступлении первого запроса от клиента в диалоге бот корректно определяет статус клиента: Новый или Существующий. Вносит новых клиентов в CRM систему. |
| **Мультиязычная работа** | Бот распознаёт и передаёт информацию на двух языках: Английский и Арабский. | Бот распознаёт информацию на двух языках: Английском и Арабском, Предоставляет ответ на языке обращения (одном из двух указанных выше). |
| **Распознавание голосовых сообщений** | Бот распознаёт голосовые сообщения и отвечает на них. | Бот распознаёт голосовые сообщения на арабском и английском языках и отвечает на них на языке обращения. |
| **Консультирование пользователей** | Консультирование пользователей по ассортименту товаров компании. | Консультирование пользователей по актуальному ассортименту товаров компании, их цене, характеристикам, срокам доставки. |
| **Реализация продажи товара**            | Реализация продажи товара исходя из приложенных правил ведения диалога. | Чат-бот реализовывает продажу товара в соответствии с приложенными правилами ведения диалога. |
| **Работа с возражениями** | Работа с возражениями клиента в соответствии с приложенными правилами и ценностями компании.  | Бот прорабатывает возражения клиентов руководствуясь методиками и ценностями компании, описанными в приложении. |
| **Предоставление клиенту статуса заказа** | При наличии заказа предоставление статуса (подтверждение, доставка, оплата) после запроса.  | Бот по запросу предоставляет корректный статус заказа. |
| **Напоминания об оплате. «Дожимы»** | Бот производит дожимы при несогласовании Sales Order. Отправляет напоминания об оплате при неуплате товара. | Бот производит дожимы при несогласовании Sales Order согласно правилам ведения диалога и описанной стратегии. Отправляет напоминания об оплате при неуплате товара. |
| **Сбор обратной связи** | После закрытия сделки бот собирает обратную связь о качестве доставки и товаре. | После закрытия сделки бот собирает обратную связь о качестве доставки и товаре. |
| **Рассчёт стоимости заказа** | Актуальный рассчёт стоимости заказа (товар+доставка) с учётом скидок и наценок. | Бот производит актуальный рассчёт стоимости заказа (товар+доставка) с учётом скидок и наценок, руководствуясь прайс-листом и предоставленными сведениями о скидках и наценках. |
| **Создание карточки сделки в CRM системе** | Создание и заполнение (имя, телефон, email, источник,UTM метка) карточки сделки в CRM системе. | Бот создаёт и корректно заполняет карточку сделки в CRM системе. |
| **Передвижение сделок в воронке** | Передвижение сделок (по алгоритму «Новый Deals» → «В работе» → «Согласование» → «Оплачен» ) в CRM системе. | Бот корректно передвигает сделки в CRM системе по указанному алгоритму и присваивает верный статус сделки. |
| **Сохранение истории переписки** | Чат бот сохраняет историю переписки с клиентами в базе данных. | В базе данных отображается история переписки чат-бота с клиентом. |
| **Оценка качества работы чат-бота продавца-консультанта** | Настройка бота контроля качества, оценивающего работу чат-бота продавца-консультанта по алгоритму. | Бот верно оценивает работу ИИ продавца в соответствии с алгоритмом.методикой ( нужно вставить в ТЗ) |
| **Предоставление отчётности по работе ИИ продавца** | Бот контроля качества по запросу предоставляет отчётность о работе чат-бота продавца-консультанта. | Бот контроля качества по запросу предоставляет отчётность о работе чат-бота продавца-консультанта. |
| **Предоставление отчётности по работе продавцов** | Бот контроля качества по запросу предоставляет отчётность о работе продавцов исходя из анализа диалогов. | Бот контроля качества анализирует диалоги продавцов и по запросу предоставляет отчётность о их работе. |

**2.2. Дополнительные требования:**

* Техническая возможность модернизации чат-бота в будущем.

* Предоставление прав администратора в чат-боте.

* Предоставить возможность редактирования промптов чат-бота. (При условии снятия ответственности за работу чат-бота с исполнителя).
