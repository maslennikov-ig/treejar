Привет! Мы продолжаем работу над проектом **Noor AI Seller** (репозиторий Treejar). 

**Контекст и текущий статус:**
Мы успешно завершили Недели 1–5. База данных, вебхуки Wazzup, RAG (pgvector), интеграции с Zoho CRM и Inventory, а также генерация PDF-коммерческих предложений (WeasyPrint) работают стабильно. Кодовая база покрыта тестами на 90% (135 тестов), Strict MyPy проходит без ошибок. Сейчас в `main` нет неслитых веток.

**Твоя задача на эту сессию — реализовать функционал "Недели 6" (Персональные цены и эскалация), а также закрыть один долг с Недели 4.**

**План работ (строго по TDD и через изолированные worktrees):**

1. **Долг Недели 4 (Context Enrichment):**
   - Настроить обогащение System Prompt историей покупок клиента из Zoho CRM (сейчас `lookup_customer` ищет контакт, но история покупок не попадает в промпт для LLM).

2. **Индивидуальные цены и скидки:**
   - Реализовать логику применения скидки на основе сегмента клиента из CRM (например, VIP = 15%, Постоянный = 10%).
   - Отразить эти персональные цены в ответах LLM (при вызове `get_stock_bulk`) и в генерации PDF-предложения.

3. **Передача диалога менеджеру (Escalation):**
   - Создать модель БД `Escalation` (статусы pending/in_progress/resolved).
   - Создать API: `POST /api/v1/conversations/{id}/escalate`.
   - Настроить LLM на распознавание 18 триггеров эскалации (просит человека, негатив, B2B заказ >10K, нестандартные условия и т.д.) и вызов соответствующего Tool для перевода на менеджера.
   - Реализовать уведомление менеджеров через Wazzup (отправка истории диалога).

4. **Follow-up и правило 24 часов (WhatsApp):**
   - Реализовать фоновую задачу (Cron job / APScheduler) для проверки `last_message_at`.
   - Если прошло >24ч — разрешить только отправку шаблонных сообщений (HSM templates).
   - Настроить расписание follow-up: 24ч → 3д → 7д.

**Правила работы (ОЧЕНЬ ВАЖНО):**
1. **Аппаратные MCP-серверы:** У тебя есть `context7` (для свежей документации по PydanticAI, FastAPI, SQLAlchemy) и `shadcn`. Всегда проверяй доки перед написанием сложного кода.
2. **Worktrees:** Используй скилл `using-git-worktrees` для создания ветки (например, `feature/week6-escalation`). НЕ пиши код напрямую в `main`.
