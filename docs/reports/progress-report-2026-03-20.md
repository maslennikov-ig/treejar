## Краткий обзор

За две прошедшие недели проект совершил **качественный скачок** — от внутренней лабораторной разработки к **работающему продукту с живыми клиентами в WhatsApp**. Бот уже принимает и обрабатывает реальные сообщения, распознаёт голосовые записи, отправляет фотографии товаров, генерирует PDF-предложения с изображениями и автоматически собирает обратную связь после продажи.

Ключевые прорывы периода:

- **Голосовые сообщения:** Бот теперь понимает голосовые записи клиентов в WhatsApp — распознаёт речь через OpenAI Audio и отвечает как на обычное текстовое сообщение.
- **Визуальный каталог в чате:** При поиске товаров клиент получает не только текстовое описание, но и фотографии продукции прямо в WhatsApp-диалоге.
- **Система пост-продажного сервиса:** Автоматический сбор обратной связи, отслеживание статуса заказа через Zoho CRM, система реферальных рекомендаций.
- **Auto-FAQ с участием менеджера:** Когда бот не знает ответа — он спрашивает менеджера в Telegram, запоминает ответ и в будущем отвечает сам.
- **Мультиязычность:** Бот автоматически переводит ответы менеджера на язык клиента (арабский, английский, русский).

## Что было сделано (недели 5–6)

### 1. Распознавание Голосовых Сообщений

Реализована полноценная поддержка голосовых сообщений в WhatsApp — одна из самых востребованных функций для клиентов из региона Ближнего Востока, где голосовой ввод является основным способом общения:

- **Speech-to-Text через OpenAI Audio:** Голосовое сообщение клиента автоматически скачивается, конвертируется и распознаётся. Текст подставляется в диалог с ботом как обычное сообщение — клиент даже не замечает разницы.
- **Эволюция модели:** Начали с open-source Voxtral, но после тестирования на реальных данных перешли на OpenAI gpt-audio-mini — значительно выше точность на арабском и английском языках.
- **Параллельная обработка:** Загрузка аудио и подготовка контекста выполняются одновременно для минимизации задержки ответа.
- **Resilient-архитектура:** При сбое распознавания клиент получает вежливое сообщение «не удалось распознать, напишите текстом», а не молчание бота.

### 2. Визуальный Каталог в WhatsApp

Бот теперь отправляет фотографии товаров прямо в чат, превращая WhatsApp в полноценный визуальный каталог:

- **Фото товаров в результатах поиска:** Когда клиент ищет мебель, бот отправляет изображения найденных товаров. Клиент видит товар, не покидая мессенджер.
- **Параллельная отправка:** Изображения отправляются одновременно через параллельные запросы к Wazzup API — клиент получает все фото практически мгновенно.
- **Фото в PDF-предложениях:** Коммерческие предложения в PDF теперь содержат встроенные изображения товаров (base64 data URI), что делает документ самодостаточным и визуально привлекательным.

### 3. Пост-продажный Сервис (Post-Sale Feedback)

Создана полная цепочка послепродажного обслуживания, которая повышает лояльность клиентов и генерирует повторные продажи:

- **Автоматический сбор отзывов:** Через настраиваемый интервал после доставки бот спрашивает клиента о впечатлениях. Оценки сохраняются в базе данных и доступны в аналитике.
- **Отслеживание статуса заказа:** Интеграция с Zoho CRM и Zoho Inventory позволяет боту в реальном времени отвечать на вопрос «Где мой заказ?» с актуальным статусом.
- **Timestamp `deal_delivered_at`:** Новое поле в CRM для точного отслеживания момента доставки и запуска фидбек-цепочки.
- **Защита от дублирования:** Unique constraint гарантирует, что клиент не получит повторный запрос обратной связи.

### 4. Auto-FAQ: Самообучающаяся База Знаний (бонус — вне ТЗ)

> **Примечание:** Данная функциональность **не входила в первоначальное техническое задание**. Мы реализовали её по собственной инициативе как бесплатный бонус, поскольку увидели, что это критически повышает ценность продукта — бот, который учится на каждом диалоге, со временем становится только умнее и требует всё меньше участия менеджера.

Реализована уникальная система, позволяющая боту расширять свои знания через взаимодействие с живыми менеджерами:

- **Telegram-интерфейс менеджера:** Когда бот не может найти ответ в базе знаний, он отправляет вопрос менеджеру в Telegram. Менеджер отвечает — и ответ сохраняется как новая FAQ-запись.
- **RAG-интеграция:** Новые FAQ-записи автоматически индексируются в векторной базе и становятся доступны для семантического поиска в будущих диалогах.
- **Нормализация на английский:** FAQ-записи сохраняются на английском для унифицированного семантического поиска, а при ответе клиенту переводятся на его язык.
- **Автоматический перевод ответов менеджера:** Если менеджер отвечает на русском, а клиент пишет на арабском — бот автоматически переведёт ответ.

### 5. Система Уведомлений и Оценки Менеджеров

Построена двусторонняя система управления качеством:

- **Telegram-уведомления:** Менеджеры получают мгновенные уведомления в Telegram при эскалациях, критических событиях и завершении сделок. Все номера телефонов маскируются (PII protection).
- **Оценка качества менеджеров (Manager Assessment):** Система автоматически анализирует работу менеджеров по структурированным критериям — скорость реакции, качество ответов, конверсия.
- **Реферальная система:** Реализована логика реферальных рекомендаций и кросс-продаж для Stage 2 бизнес-процессов.
- **Аналитические отчёты:** Модуль rejection & conversion analytics отслеживает причины отказов и конверсию на каждом этапе воронки.

### 6. Миграция на BAAI/bge-m3 (Sentence-Transformers)

Проведена стратегическая миграция движка семантического поиска:

- **С fastembed на sentence-transformers:** Переход на библиотеку `sentence-transformers` с моделью BAAI/bge-m3 — мультиязычная модель с поддержкой 100+ языков, критически важно для арабского рынка.
- **Умная синхронизация эмбеддингов:** Embeddings пересчитываются только при реальном изменении контента товара (название, описание, характеристики), а не при каждой синхронизации — экономия вычислительных ресурсов до 90%.
- **Автоматический lifecycle:** При добавлении нового товара в Zoho — эмбеддинги создаются автоматически. При удалении — очищаются. Полный автоматизм без ручного вмешательства.

### 7. WhatsApp-форматирование и Стабилизация

Масштабная работа по адаптации LLM-выхода под ограничения WhatsApp:

- **Конвертация Markdown в WhatsApp-формат:** LLM генерирует ответы в Markdown (заголовки, таблицы, code blocks), а специальный форматтер преобразует их в WhatsApp-совместимый формат (жирный, курсив, моноширинный текст).
- **Таблицы → читаемый текст:** Markdown-таблицы автоматически конвертируются в структурированный текстовый формат, удобный для чтения на мобильном экране.
- **WhatsApp formatting rules в system prompt:** LLM изначально получает инструкции о правилах форматирования WhatsApp, что снижает потребность в пост-обработке.

### 8. Безопасность и DevOps

Проведён комплексный аудит безопасности и реорганизация инфраструктуры:

- **IP Allowlist вместо Bearer Token:** Webhook-аутентификация переведена с Bearer-токена на IP-whitelist Wazzup-серверов — более надёжная защита от подделки запросов.
- **Timing-safe сравнение:** Все проверки API-ключей используют `hmac.compare_digest` для защиты от timing-атак.
- **PII-маскирование:** Телефонные номера маскируются в логах и уведомлениях (+971****1234).
- **Аудит CR-S2:** 10 исправлений по результатам аудита безопасности: авторизация endpoint'ов, защита логов, паттерны аутентификации.
- **Раздельные Prod/Dev на VPS:** Production и Development среды полностью изолированы в отдельных директориях с разными Docker Compose проектами — исключены коллизии контейнеров.
- **Единый CI/CD Pipeline:** CI и Deploy объединены в единый GitHub Actions workflow — автоматическое тестирование, сборка и развёртывание в одном процессе.
- **Structured Logging + Logfire:** Внедрено структурированное логирование с интеграцией Logfire для мониторинга и отладки в production.

### 9. Качество Кода и Тестирование

Кодовая база достигла высочайших стандартов качества:

- **416 тестов в 78 модулях** — почти удвоение за 2 недели (было 220 / 47).
- **11 интеграционных верификационных скриптов** + ручной чеклист тестировщика для проверки всех функций на production.
- **Bot Test Suite:** Специализированный инструмент для прямого API-тестирования бота — эмуляция реальных диалогов без запуска полного стека.
- **0 ошибок Mypy** в 179 файлах — полная строгая типизация без исключений.
- **0 ошибок Ruff** — чистый код по всем правилам линтера.
- **Тестового кода больше, чем production-кода:** 11 585 строк тестов vs 10 341 строк бизнес-логики — показатель зрелости проекта.

## Текущее состояние системы

Полная цепочка обработки клиентского обращения работает end-to-end:

```
Клиент в WhatsApp
    │
    ├─ Текстовое сообщение ──→ Wazzup Webhook ──→ ИИ-обработка ──→ Ответ
    ├─ Голосовое сообщение ──→ Speech-to-Text ──→ ИИ-обработка ──→ Ответ
    ├─ «Покажи стулья» ──→ RAG-поиск + фото ──→ Карточки товаров в чате
    ├─ «Сделай предложение» ──→ PDF с фото ──→ Документ в WhatsApp
    ├─ «Где мой заказ?» ──→ Zoho CRM/Inventory ──→ Статус доставки
    ├─ Непонятный вопрос ──→ Менеджер в Telegram ──→ Ответ + FAQ
    └─ После доставки ──→ Автоматический фидбек ──→ Оценка + аналитика
```
